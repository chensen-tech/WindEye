# 06 — 业务应用系统

## 6.1 模块概述

将底层 KG 和 Multi-Agent 能力转化为业务人员可操作的界面，实现"数据→知识→分析→决策→行动"的完整闭环。

---

## 6.2 应用一：风险传递可视化看板

### 6.2.1 功能概述

基于 AntV G6 的知识图谱可视化看板，支持分层展示、交互式探查和风险路径追踪。这是业务人员与知识图谱交互的主界面。

### 6.2.2 核心功能

#### 分层展示

```
┌──────────────────────────────────────────┐
│  [图层切换]  ☑ 主体层  ☑ 事件层          │
│              ☑ 特征层  ☑ 法规层          │
├──────────────────────────────────────────┤
│                                          │
│        ┌──────────┐                      │
│        │ 法规节点  │  ← 第4层 (顶部)      │
│        └─────┬────┘                      │
│              │ 关系边                     │
│        ┌─────┴────┐                      │
│        │ 特征节点  │  ← 第3层             │
│        └─────┬────┘                      │
│              │                           │
│        ┌─────┴────┐                      │
│        │ 事件节点  │  ← 第2层             │
│        └─────┬────┘                      │
│              │                           │
│        ┌─────┴────┐                      │
│        │ 主体节点  │  ← 第1层 (底部)      │
│        └──────────┘                      │
│                                          │
└──────────────────────────────────────────┘
```

**要求：**
- 四个图层可独立开关显示/隐藏
- 每层颜色区分（如：主体-蓝、事件-橙、特征-红、法规-紫）
- 支持图层间的垂直分层布局（力导向 + 纵向分层约束）

#### 交互操作

| 操作 | 行为 | 技术要求 |
|------|------|---------|
| **节点点击** | 展开节点详情面板 (属性列表 + 1跳邻居) | < 200ms 响应 |
| **节点下钻** | 以当前节点为中心，展开 N 跳邻居 | 默认 1 跳，可配置 2-3 跳 |
| **关系高亮** | 鼠标悬停关系边，高亮显示关联节点和路径 | 高亮 + 其他节点半透明 |
| **路径追溯** | 选中两个节点，自动查询并高亮所有路径 | 调用 `/kg/path` API |
| **框选/圈选** | 批量选中节点，支持批量标记/导出 | G6 框选交互 |
| **子图导出** | 将当前可视区域导出为 PNG/SVG 图片 | 用于嵌入报告 |
| **风险滤镜** | 按风险等级筛选节点 (高/中/低/无) | 颜色 + 筛选联动 |
| **全文搜索** | 搜索框输入实体名称，自动定位 | 模糊匹配 + 自动居中 |

#### 画布控制

- 缩放 (0.1x ~ 5x)
- 全图适配 (FitView)
- 撤销/重做 (操作历史栈，深度 20)
- 布局切换 (力导向 / 分层 / 环形 / 网格)

### 6.2.3 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 图渲染引擎 | AntV G6 v4 | 支持自定义节点/边、布局算法、交互插件 |
| 3D/大规模图 | react-force-graph-2d | 力导向布局的大型图渲染 |
| 节点样式 | 自定义 G6 Node + 图片节点 | 支持企业 Logo、风险图标 |
| 数据流 | WebSocket (实时更新) + REST (查询) | 图谱变更实时推送前端 |

## 6.3 应用二：社区报告自动生成

### 6.3.1 功能概述

调用 Multi-Agent 推理引擎，自动生成包含图谱分析、风险研判、处置建议的结构化报告。

### 6.3.2 触发方式

| 方式 | 描述 | 配置 |
|------|------|------|
| **定时触发** | 每日/每周定时对重点监控实体生成报告 | Cron 配置 |
| **事件触发** | 监测到新增高风险事件时自动生成 | 事件监听 + 阈值 |
| **手动触发** | 用户在看板中选择实体，点击"一键生成报告" | UI 按钮 |
| **批量触发** | 选择实体列表批量生成 | 异步队列 |

### 6.3.3 报告内容

| 章节 | 内容来源 | 输出格式 |
|------|---------|---------|
| 封面 | 系统配置 | 标题 + 日期 + 报告编号 |
| 摘要 | Reporter Agent | 200 字以内核心结论 |
| 关联网络 | Retriever Agent + G6 截图 | 图谱可视化 + 表格 |
| 风险路径 | Analyst Agent | 路径图 + 文字说明 |
| 异常发现 | Analyst Agent + 异常检测 | 列表 + 证据 |
| 合规研判 | Compliance Agent | 法条引用 + 违规认定 |
| 处置建议 | Reporter Agent (汇总) | 优先级排序列表 |
| 附录 | 系统 | 数据来源、免责声明 |

### 6.3.4 导出格式

- **在线查看：** 浏览器中渲染的 HTML 报告 (ProLayout + 组件化)
- **导出格式：** PDF (Puppeteer 服务端渲染) / Word (docx 模板) / Markdown

## 6.4 应用三：协同治理社区报告

### 6.4.1 功能概述

发现风险后，系统汇总风险主体、相关群体、风险传导路径、异常发现和合规匹配结果，生成协同治理社区报告。报告用于辅助分析人员快速理解风险社区结构和治理优先级。

### 6.4.2 报告结构

```
风险主体识别 → 群体发现 → 风险传导路径 → 合规分析 → 协同治理社区报告
```

### 6.4.3 报告字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `report_id` | String | 唯一报告编号 |
| `executive_summary` | String | 社区治理结论摘要 |
| `related_entities` | Array[Entity] | 关联风险主体 |
| `communities` | Array[Community] | 群体发现结果 |
| `risk_paths` | Array[Path] | 风险传导路径 |
| `compliance_matches` | Array[Compliance] | 合规匹配结果 |
| `governance_plan` | Object | 治理建议、优先级和升级规则 |
| `recommendations` | Array[String] | 建议措施 |
| `created_at` | DateTime | 生成时间 |

### 6.4.4 治理建议类型

| 建议类型 | 说明 |
|---------|------|
| **风险核查** | 对关键主体、资金链路和股权关系进行重点核查 |
| **合规补充** | 补充披露、关联关系说明或内部决策材料 |
| **持续观察** | 跟踪风险传导路径上的主体状态变化 |
| **升级研判** | 对高风险路径触发进一步研判和人工复核 |

## 6.5 应用四：系统管理

| 功能模块 | 子功能 |
|---------|--------|
| **用户管理** | 用户 CRUD、角色分配、权限配置 |
